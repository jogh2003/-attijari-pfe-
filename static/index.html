Comment UiPath traite un nouveau ticket
Flux de détection
  • CheckAlerte.xaml interroge GET /api/alertes?seuil=0.75 et analyse le ticket en moins de 10ms.
  • Si le score est ≥ 0,75, une alerte active est créée et UiPath lance la chaine de robot.
  • Score ≥ 0,85 : priorité 1, le robot exécute automatiquement notification et résolution.
  • Score 0,75–0,85 : priorité 2, NotifierIT prévient le responsable IT avant clôture.
Rôles UiPath
  • CheckAlerte.xaml : détecte les anomalies et déclenche les alertes.
  • NotifierIT.xaml : notifie le responsable IT pour les alertes critiques.
  • ConfirmerResolution.xaml : clôture l’alerte après validation IT.
  • Responsable IT : reçoit les alertes, valide les actions et clôture les tickets.
